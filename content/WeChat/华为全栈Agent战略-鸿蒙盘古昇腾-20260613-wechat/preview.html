被制裁逼出来的全栈——华为 HDC 2026 背后的 Agent 棋局

6月12日的东莞松山湖，余承东站上 HDC 2026 舞台，一口气扔出了三件事：开源大模型 openPangu 2.0、全场景智能操作系统 HarmonyOS 7、以及昇腾算力的亲和优化方案。

如果你只把它们当三个独立产品看，那大概率会得出"华为又在追参数"的结论。但把这三件事叠在一起，画面就变了——这是华为从芯片到模型到操作系统的一盘全栈棋。而且这盘棋，不是主动选的，是被逼出来的。

🚫 没卡，就只能换打法

先看最底层。余承东在发布会上说了句老实话：华为的昇腾算力，大量拿去支持国内其他企业的需求了，自己留的数量非常有限。

这不是谦虚。从 2019 年实体清单开始，TSMC 断供、NVIDIA GPU 禁运、HBM 内存受限、EDA 工具禁令——能卡脖子的环节基本都卡了一遍。华为手上的牌，只有中芯国际 N+2 工艺产出的昇腾和麒麟芯片，制程水平大概相当于 7nm，跟台积电 3nm 差了两代。

所以 openPangu 2.0 不走参数竞赛路线。Pro 版 505B 总参数、18B 激活参数，Flash 版 92B 总参、6B 激活——放在业界预期中的下一代大模型（GPT-5、Claude 4 普遍被认为在万亿参数级别）旁边，数字不算大。但华为走的是 MoE（混合专家）架构：训练时 505B 的"专家委员会"各司其职，推理时每次只激活 18B，算力消耗跟一个 18B 的小模型差不多。

单卡吞吐率做到业界主流开源模型的 2 倍，512K 上下文窗口。说白了，就是在算力受限的前提下，把每一张昇腾芯片的利用率榨到极致。我认识的做推理部署的团队里，已经有人在昇腾上跑通了 vLLM，反馈是"能用，但调优确实要花时间"——对于一套正在追赶 CUDA 的软件栈来说，这个评价不算差。

开源策略更激进

这还没完。CANN（华为的计算架构，相当于 NVIDIA 的 CUDA）在 2025 年底正式开源了。现在开发者通过 torch_npu 适配器，把代码里的 .cuda() 换成 .npu() 就能跑。社区里 vLLM 的昇腾插件也上线了。

DeepSeek V4 已经在 1000 张昇腾 910C 上完成了后训练，并在昇腾全系验证了推理——虽然预训练可能还是用了 NVIDIA 的卡，但这是国产芯片第一次支撑起顶级开源模型的完整后训练流程。

6 月 30 日起，openPangu 2.0 的 7 大组件将陆续开源。多说一句，业界通常开源的只有 4 项——模型结构、权重、技术报告、推理代码；华为额外多开源了 3 项：预训练代码、后训练代码、训推算子。相当于把完整训练链路都交出来了。

这条效率路线能不能走到最后是另一个问题，但至少方向是清晰的：没有暴力 scaling 的资本，就只能在架构和工程上找答案。

🤖 一个操作系统，开始替用户干活

往上走一层，HarmonyOS 7 是这次 HDC 最值得琢磨的发布。

过去几年手机系统的竞争，基本围绕着谁更流畅、谁应用更多、谁拍照更好。大模型火了之后，各家开始往系统里塞 AI 助手、AI 修图、AI 摘要。但 HarmonyOS 7 做了一件不一样的事：它把操作系统的底层架构，以 Agent 为核心理念重写了一遍。

鸿蒙智能体框架 HMAF 2.0 提出"意图即服务"——用户说出目标，系统理解任务，然后自己去调度合适的应用能力。复杂任务的执行成功率做到 90% 以上。开放了 20 多项系统级 AI 能力，甚至首次向开发者开放了 GUI 操控权限。

小艺也不是以前那个"帮我定个闹钟"的语音助手了。现在的数据是：日活 1.8 亿用户，日均唤醒 30 亿次，接入了 2100 多项鸿蒙系统能力、500 多个伙伴精选 Skill、2000 多个鸿蒙智能体。它可以调用日程、健康数据、运动记录等 200 多项系统级感知数据，跨设备完成任务——比如你在手机上说要跑马拉松，它能结合你的身体数据、日程安排、天气信息，生成训练计划，再同步到手表上。

⚡ Agent 不是 ChatBot

这跟"手机上装个 ChatBot"是两回事。前者是用户自己打开一个个 App 手动操作，后者是系统理解意图后替你调度资源。方舟引擎首次内置了"性能大模型"，基于 20 多亿条场景数据做智能流畅调度——相比 HarmonyOS 6 性能提升 15%，应用切换快 25%，重点应用保活率提升 34%。

说真的，30B 端侧模型能在麒麟上跑出这个效果，我确实有点意外。这个秋天，30B 总参数、2B 激活参数的端侧盘古模型将在麒麟芯片上运行。通过量化剪枝、专家复用和激活专家预测三项优化，推理吞吐率比上一代提升 5 倍以上。这意味着手机上的 Agent 可以完全不依赖云端完成很多任务——隐私、延迟、离线可用性，三个问题同时解决。

🏆 唯一一个三者都自己做的

把三层叠在一起看，华为的"全栈"不是营销话术。

OpenAI 有最强的模型，但没有设备。ChatGPT 的全球用户量早已是现象级——但用户再多，它也卡在浏览器和 API 里，触碰不到操作系统的底层权限。

Apple 有设备、有芯片、有系统，但 AI 模型是目前四家里最弱的。Siri AI 在 WWDC 2026 上秀了 App Intents 和多步推理能力，但跟业界最顶尖的闭源模型比，差距还是肉眼可见。

Google 看起来最接近全栈——Android 有全球近 80% 的份额，Gemini 2.5 的推理能力也很强。但 Android 的硬件是碎片化的：高通的骁龙、联发科的天玑、三星的 Exynos——Google 不造手机芯片（Tensor 本质上是三星代工加 ARM 公版），也没法像华为那样从芯片指令集一直优化到操作系统调度。

华为是唯一一个三者全握在自己手里的玩家：昇腾芯片（硬件）→ 盘古大模型（AI）→ 鸿蒙系统（OS）。更重要的是，这三层不是简单拼在一起的。

昇腾的亲和优化让盘古在同规格硬件上跑出更高的吞吐率；盘古的 MoE 架构和 512K 上下文为鸿蒙的 Agent 任务提供了推理基础；鸿蒙的 Agent 亲和架构反过来为模型提供了系统级的上下文和权限。

⚡ 三层联动

Agent 能调度系统资源（比如读取日程、控制应用），系统能在端侧跑推理（隐私加离线），硬件、模型、系统之间有协同优化的空间——不是各做各的。

📊 505B 的天花板在哪

但有一个问题绕不过去。余承东自己说了，算力有限。

505B 的参数规模，在当前的大模型军备竞赛里确实不算大。GPT-5 传闻是万亿级别，Claude 4 也在这个量级。MoE 架构虽然推理效率高，但模型能力的上限毕竟和总参数量相关。

全栈策略到底是华为的主动布局，还是制裁逼出来的被动应对？这个问题的答案很可能是：两者都有。制裁确实卡住了昇腾芯片的制程迭代，让堆参数这条路走不通。但华为在芯片互联、软件优化、系统架构上的投入，也不是最近一两年才开始的——2018 年昇腾路线图就发布了，2019 年鸿蒙立项，2021 年盘古 1.0 就已经是全球最早的大模型之一。

余承东在发布会上说："在我余承东的字典里，没有第二，只有第一。"他 2025 年国庆前夕被重新任命负责大模型业务，等于华为把最会打仗的人派到了一线。至于效率路线能走多远，关键看两点：一是昇腾芯片的自研 HBM 和先进封装能不能量产突破，二是开源的盘古生态能不能吸引足够多的开发者。

🔮 Agent 的终局不在云端

AI Agent 这场仗打到现在，一个趋势越来越明显：真正的 Agent 不能只活在云端 API 里。

云端模型再强，也解决不了延迟、隐私、离线这三个硬问题。你让一个 Agent 帮你订机票、回微信、安排日程，它如果每次都要绕一圈云端再回来，体验一定不如直接在手机上完成。

华为押注的方向是端侧——30B 的模型跑在麒麟芯片上，鸿蒙系统作为 Agent 的"运行时"，昇腾芯片提供云端协同的推理能力。这个方向不一定能赢，但至少指出了一个终局：最自然的 Agent，不是你在对话框里打字，而是系统在理解你、替你干活。

HarmonyOS 6 终端设备已经突破 6600 万，注册开发者 1100 多万，应用市场 40 多万款应用和服务。这个生态规模跟 iOS 和 Android 比还有差距，但如果 Agent 真的改变了人机交互的入口——从"找 App"变成"交代任务"——那鸿蒙可能跳过一部分传统应用生态的追赶逻辑。

当然，这些都还是如果。

HarmonyOS 7 开发者 Beta 刚发布，正式版要等秋季 Mate 90 系列首发。openPangu 2.0 要到 6 月 30 日才开始开源。小艺能不能真的把 90% 以上的复杂任务执行到位——说实话，发布会上说的数字和实际体验之间通常有个 gap——还要看第一批用户的真实反馈。

但有一件事是可以确定的：当 AI Agent 成为新入口，操作系统的价值正在被重新定义。华为这次发布会，把所有筹码都压在了这个方向上。

华为押注的方向是端侧——30B 的模型跑在麒麟芯片上，鸿蒙系统作为 Agent 的"运行时"，昇腾芯片提供云端协同的推理能力。这个方向不一定能赢，但至少指出了一个终局：最自然的 Agent，不是你在对话框里打字，而是系统在理解你、替你干活。

HarmonyOS 6 终端设备已经突破 6600 万，注册开发者 1100 多万，应用市场 40 多万款应用和服务。这个生态规模跟 iOS 和 Android 比还有差距，但如果 Agent 真的改变了人机交互的入口——从"找 App"变成"交代任务"——那鸿蒙可能跳过一部分传统应用生态的追赶逻辑。

当然，这些都还是如果。

HarmonyOS 7 开发者 Beta 刚发布，正式版要等秋季 Mate 90 系列首发。openPangu 2.0 要到 6 月 30 日才开始开源。小艺能不能真的把 90% 以上的复杂任务执行到位——说实话，发布会上说的数字和实际体验之间通常有个 gap——还要看第一批用户的真实反馈。

但有一件事是可以确定的：当 AI Agent 成为新入口，操作系统的价值正在被重新定义。华为这次发布会，把所有筹码都压在了这个方向上。
