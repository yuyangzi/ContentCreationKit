## 一、Grill-Me 审核问答记录

### Q1：目标读者定位

**问**：主题列出三类读者（AI 开发入门者、转型工程师、在校学生），起点完全不同，一个路线图无法同时服务好三类人。主攻谁？

**答**：主攻 **想转型 Agent 开发的工程师**（已有编程基础、了解 HTTP/API、用过至少一个 LLM）。学生和入门者为次要受众，需在文章开头标注前置条件（Python 基础、API 调用经验）。

---

### Q2：路线图 vs 教程

**问**：四个阶段覆盖了一个完整课程体系的内容量，一篇公众号文章（3000-8000 字）不可能承载教学级深度。你是做纯路线图（A）还是深扎一个阶段当教程（B）？

**答**：**系列文章模式**——每篇深扎一个阶段做教程，多篇覆盖全部四阶段。

---

### Q3：系列结构

**问**：总纲 + 4 深扎（5 篇），还是纯 4 深扎（4 篇），还是混合（4 篇）？

**答**：**A 方案——1 篇总纲 + 4 篇深扎**，共 5 篇。总纲独立发路线图，四篇深扎各讲一个阶段。

---

### Q4：总纲篇的独立价值

**问**：总纲只有 2000 字的内容量，凭什么独立成篇？行业分析（A）、学习哲学（B）、还是就业导向（C）？

**答**：**A+B 混合**——行业分析做序（为什么现在学）+ 学习哲学做核（为什么这么学）+ 路线图做收（学什么），就业数据作为点缀穿插。

---

### Q5：总纲与深扎的内容切割线

**问**：总纲对每个阶段描述到什么深度？是只给导航，还是也给内容摘要？

**答**：总纲只讲到**"是什么 + 为什么不能跳过 + 一句核心原理"**。代码、详细原理、设计原则全部留给深扎篇。总纲的价值在"判断力"和"路径选择"。

---

### Q6：阶段一的内容重心

**问**：阶段一（LLM 基础 + Prompt + Function Calling），对转型工程师来说重心该放哪？砍掉 Transformer/MoE 细节，主攻 Function Calling？

**答**：加入**上下文工程、Harness Engineering、Agent Loop** 三个最新概念。最终布局：

| 模块 | 占比 |
|------|:--:|
| LLM 基础 | 20% |
| 上下文工程 | 25% |
| Function Calling + Agent Loop | 40% |
| Harness Engineering（驾驭工程） | 15% |

Harness Engineering 定义："Human Steer, Agent Execute"——设计约束机制、反馈回路和受控运行环境的系统工程实践。

---

### Q7：教程代码用什么框架

**问**：LangChain 还是原生 SDK？2026 年 LangChain 仍在学习场景占优，但生产团队在向厂商 SDK 迁移。

**答**：**OpenAI SDK 原生教学**——暴露底层机制（tool_choice、finish_reason、tool_call_id），读者能看到完整流转。末尾补一段 LangChain `@tool` 装饰器的 5 行等价写法——"你现在知道它在底层干了什么"。

---

### Q8：教程代码跑在哪个模型上

**问**：Claude Fable 5（海外门槛高）、DeepSeek V4-Pro（国内友好）、Qwen 3.6（开源），选哪个？

**答**：**DeepSeek V4-Flash**——兼容 OpenAI SDK、国内注册无障碍、便宜（比 V4-Pro 更轻量）。标注替换 `base_url` 即可切换到其他模型。

| 仓库 | Stars | 阶段一 | 阶段二 | 阶段三 | 阶段四 |
|------|------:|--------|--------|--------|--------|
| Hello-Agents | 58,900 | 第1-3章：Agent+LLM基础 | 第4-7章：LangGraph+框架 | 第8-12章：MCP+多Agent+RL | 第13-16章：实战+毕业设计 |

### 2.2 MCP 协议验证：基本准确

| 时间节点 | 事件 | 数据 |

| 2024.11 | Anthropic 开源 MCP | ~200 万月下载 |

| 2025.07 | 微软集成 MCP 到 Copilot Studio | 4500 万月下载 |
| 2025.12 | Anthropic 将 MCP 捐赠给 Linux 基金会；OpenAI/Block 联合创始 | — |

**⚠️ 需注意的 Caveat**：

- 企业采纳率约 28%（财富 500 强），离"通用基础设施"还有距离

### 2.3 LangChain vs 厂商 SDK：需限定表述

| 维度 | LangChain | 厂商 SDK |
|------|-----------|----------|
| GitHub Stars | 139,216 | OpenAI Agents SDK: 27,130 |
| PyPI 月下载（2026.06） | ~3.0 亿 | OpenAI: ~3.2 亿 |
| 学习场景 | ✅ 首选（资源最多） | 🔶 需理解底层 |
| 生产场景 | 🔶 抽象税 + 调试难 | ✅ 团队在迁移 |

**2026 年共识**：
- 原型设计 → LangChain
- 生产部署 → 厂商 SDK 或 LangGraph（复杂有状态工作流）
- 多模型灵活性 → LangChain 或 Pydantic AI
- RAG 重负载 → LlamaIndex

**来源**：Ravoid "The LangChain Exit" (2026.04)、AI-Buzz "LangChain vs OpenAI: New PyPI Data" (2026.03)、RoboRhythms "LangChain Is Quietly Losing Developers" (2026.04)
