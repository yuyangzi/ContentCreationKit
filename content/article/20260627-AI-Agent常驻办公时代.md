# AI Agent 不再"随叫随到"：三周内，三大阵营指向同一个答案

6 月 25 日，OpenAI 联合哥大、沃顿、杜克发布了一份 Codex 经济报告。其中一组数据值得注意：Codex 用户中，25.6% 委派过等效 8 小时以上人工劳动的任务——朝九晚五整一天的工作量。还有 70.2% 委派过超过 1 小时人工的任务。

25.6% 的用户已经把 Agent 当全职人力在用。不是辅助工具，不是提效插件——是一个人。

同一天，OpenAI 的博客标题是"How Agents Are Transforming Work"。但真正值得注意的不是 OpenAI 说了什么，而是这份报告跟另外两件事撞在了一起：Claude Tag 在 6 月 23 日发布，微软 Scout 和 GitHub Copilot 桌面应用在 6 月 2 日亮相。

三周内，Anthropic、微软、OpenAI——三家路线完全不同的公司——释放了同一个信号：**AI Agent 正在从"你叫它它才来"变成"它一直在那儿"。**

这次的升级不是"更聪明了"，是身份变了。

---

## 定义"同事"需要四件东西

"AI 是同事"这个说法不算新。ChatGPT 刚出来时也有人这么说。但你回想一下，你让 ChatGPT 帮你写个邮件、查个资料——这叫同事吗？这叫工具。它没有自己的工位，没有自己的账号，不知道你昨天干了什么，也不会主动找你。

真正的同事需要四件东西：

**持久在场。** 不是打开即用、用完即走的工具，而是常驻在工作环境里，你知道它一直在。

**独立身份。** 有自己的账号、权限、可追溯的操作记录。在组织里是一个可归属的实体，不是一个共享服务号。

**主动介入。** 不等你叫它，自己发现有什么该做的，然后动手。

**组织记忆。** 跨会话积累上下文，越来越了解这个团队在干什么、做过什么决定、为什么这么做。

按这个标准，缺一项就只是"更好用的工具"。四项全满足，才算迈过了那道门槛——从你用它变成它跟你一起做事。

---

## Claude Tag：Slack 里来了一位新同事

6 月 23 日，Anthropic 发布了 Claude Tag——一个常驻 Slack 频道的 AI。它的核心设计逻辑不是"回答你的问题"，而是**像一个真人员工一样在场**。

你不 @ 它，它也会介入。团队在频道里讨论某个功能的进度，Tag 发现上周有人提过类似的需求，主动把那条消息翻出来；有人在讨论一个 Bug，Tag 直接去看日志，然后把排查结果贴进频道。Anthropic 管这叫 Ambient 模式——"环境感知式的工作"。说白了就是不请自来。

它有自己的账号。在 GitHub 上是独立的 App，在数据库里有自己的服务账号。做的事情可归属、可追溯——跟一个人干的没什么区别。

它会学习。在这个频道待久了，它知道团队的决策历史、项目的技术栈、每个人的角色分工。不用每次跟它说话都重新交代背景。

Anthropic 产品负责人 Cat Wu 给了一个数字：产品团队 65% 的代码由内部版 Tag 生成。"生成"这个词不是"辅助"——Tag 是主要编码者。非工程团队也在用它追踪产品指标、处理支持工单、诊断 Bug。

对照四要件：持久在场（常驻频道）、独立身份（独立 App 和账号）、主动介入（Ambient 模式）、组织记忆（学习频道历史）——Claude Tag 四项全占。它在 Slack 里就是一个有名字、有权限、会主动找你的同事。

---

## Scout：AI 在组织架构里有了"工号"

比 Tag 早三周——6 月 2 日，微软在 Build 2026 上发布了 Scout。

Scout 的定位词是"从 Copilot 到 Autopilot"。Copilot 是副驾驶，你在开它在旁边帮你；Autopilot 是你放手让它开。微软自己说这是个新品类：不是 Copilot Chat，不是 Copilot Cowork，是独立运行的个人 Agent。

Scout 拥有独立的 Entra ID——微软的企业身份认证体系。这意味着 Scout 在企业组织目录里是一个正式身份，有自己的工号、权限组、访问策略。不是那种"大家共用 admin 账号"的东西。每个 Scout 实例是可归属的实体。

它在后台持续运行，跨 Teams、Outlook、OneDrive、SharePoint，主动协调跨时区会议、标记风险事项、准备会议材料。Work IQ 系统学习用户的工作模式和偏好——不需要你设置规则，它自己观察。

与此同时，GitHub 也发布了 Copilot 桌面应用。从 IDE 插件升级为独立桌面应用，核心卖点是多 Agent 并行：每个会话拥有独立的 git worktree，一个写前端、一个写后端、一个跑测试，互不干扰。Canvas 面板让人和 Agent 在同一表面上协作。

两个产品放在一起，微软画出的图景很清晰：未来的企业工作流不是"人 + AI 助手"，而是"人 + AI 团队"。人从执行者变成管理者，Agent 从工具变成成员。

而 Scout 的独立 Entra ID 是这里面最被低估的一步。AI 在组织架构里正式有了身份——这不只是一个产品功能，更暗示了组织设计的底层规则在变。

## Codex 报告：不需要产品定义，行为已经给出了答案

Claude Tag 和 Scout 回答的是"AI 可以是什么"。Codex 经济报告回答的是另一个问题——"用户已经在把它当什么用了"。

这份报告由 OpenAI 研究员 Drew Johnston 联合哥伦比亚商学院、沃顿商学院和杜克大学的学者共同发布。不是产品发布会上的愿景，是硬数据。

时间拉回到 2025 年 8 月。那时候 Codex 还是一个程序员工具。到 2026 年 6 月，非开发者个人用户的周活跃用户数增长了 137 倍。公司组织的非开发者增长更夸张——189 倍。

137 倍。这不是"慢慢增长"的节奏，是某个东西突破了临界点之后的爆发。

这些非开发者在用 Agent 干什么？写报告、做分析、管理日程、处理邮件、跟踪项目进度——以前需要跨好几个工具、跟好几个人沟通才能完成的事情，现在委托给一个 Agent 就结束了。

25.6% 的用户每天委派 8 小时以上，70.2% 每天委派 1 小时以上。这意味着它们不是在"帮"人工作——这些东西正在吃掉完整的人工工作量。

Codex 报告在四要件框架里缺了三项——它本身不是一个"有身份、会主动、能记忆"的产品。但它补了最关键的一块证据：用户行为上，已经把 Agent 当全职人力了。产品定义的是上限，数据证明的是现状。

## 人的角色变了：不是"使用 AI"，是"管理混合人机团队"

到这里，三组信号都摆完了。但只看"AI 更厉害了"就错过重点了。

此前讨论过：AI 压缩了执行力、放大了判断力；AI 的决策半径在变大；Loop 范式把判断的执行部分也工程化了。这三篇讨论的是**能力层**——AI 能干什么，人的边界往哪退。

这篇文章讨论的是**关系层**——AI 是你的什么。

当你的 Slack 频道里有个东西拥有独立账号、记住你三个月的聊天记录、不等你吩咐就自己动手——你跟它的关系已经从"使用"变成了"共事"。它不是你打开的工具，是你团队里一个不睡觉的成员。

这意味着人的角色在换轨。以前你是一个"使用者"——学习怎么用好 AI、怎么写 Prompt、怎么让它的输出更符合预期。接下来你是一个"管理者"——分配任务、设定标准、审核产出、协调你的人机混合团队。

这个新角色需要什么能力？

判断力还在——而且更重要了。当 AI 给出十个方案，你选哪一个？不是选"最正确的"，是选"在这个上下文里最合适的"。这种判断没法自动化，因为它依赖的东西藏在组织记忆、团队文化和人的直觉里。

但协调能力呢？追踪进度、分配任务、提醒截止日期——这些管理动作，Agent 自己就能干。Scout 已经在做了：协调跨时区会议、标记风险、准备材料。传统意义上的"项目管理"，可能是下一批被吃掉的技能。

这跟前三篇的叙事连上了——人每次放弃一个旧身份，捡起一个新身份，然后新身份的某些部分又会被下一波自动化吞掉。

---

当一个团队里有个成员写 65% 的代码、参加每一次讨论、处理大部分执行工作——你到底是什么角色？

这个问题没有标准答案。但找到它的过程，或许就是这一轮 AI 浪潮里最值得关注的事。
