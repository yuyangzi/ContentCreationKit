## 变更清单

### 变更 1（§没有 Hook 的时代） — 移除重复主语 [allowed: redundancy]

**分类**：冗余

**位置**：第二个事故项"工具调用静默失败"段

**Before**：
```
Agent 调用了一个搜索 API，API 返回了空结果或者部分结果，Agent 没检测到异常
```

**After**：
```
Agent 调用了一个搜索 API，返回了空结果或者部分结果，Agent 没检测到异常
```

**理由**：前句主语已明确是"搜索 API"，无需重复"API"作为第二个分句主语。

---

### 变更 2（§第一批解法） — 弱化宣传性修饰 [allowed: promotional-language]

**分类**：措辞调整

**位置**：回调示例代码之后，总结性段落

**Before**：
```
这在当时解决了一个巨大的痛点：你终于能"看见" Agent 内部在干什么了。
```

**After**：
```
这在当时解决了一个实在的痛点：你终于能"看见" Agent 内部在干什么了。
```

**理由**："巨大的"属于 Humanizer §4 标记的宣传性语言（promotional language）。"实在的"保持强调的同时更克制，符合 StyleRule §2"弱化绝对"原则。

---

### 变更 3（§把 Hook 用好：五条实践准则） — 简化冗余表述 [allowed: redundancy]

**分类**：冗余

**位置**：五条实践准则的引言段

**Before**：
```
这些准则来自生产环境中反复验证过的模式，不是理论推演。
```

**After**：
```
这些准则来自生产环境的反复验证，不是理论推演。
```

**理由**："生产环境中……的模式"存在概念嵌套（"模式"本身就隐含环境），简化为"生产环境的反复验证"直接传达信息来源，且与后半句"不是理论推演"形成更整齐的对仗。

---

### 变更 4（§把 Hook 用好：五条实践准则） — 消歧主语 [allowed: clarity]

**分类**：澄清

**位置**：第五准则末尾

**Before**：
```
这不是 bug，是你自己编排的策略链。
```

**After**：
```
这不是框架的 bug，是你自己编排的策略链。
```

**理由**：原句"这"的指代在上下文中有歧义——读者可能不确定"这"指"原始手机号泄露到日志系统"还是"中间件组合的顺序行为"。加"框架的"明确排除"系统 bug"的暗示，强调策略编排者的责任。
