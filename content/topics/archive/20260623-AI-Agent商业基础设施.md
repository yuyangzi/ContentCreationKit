### 第一章：支付层 — 蚂蚁 AMP 协议

> 关键词：智能体怎么付钱？谁来为 Agent 的行为兜底？

**核心叙事**：
- AMP（Agentic Mobile Protocol）做了什么：将 Identity + Authorization + Payment + Settlement + Trust 统一纳入移动支付生态
- KYA（Know Your Agent）：为智能体建立统一的信任评级标准，解决"我怎么知道这个 Agent 是可信的"
- AgentSafePay：蚂蚁"敢付敢赔"扩展到 Agent 交易场景
- 全球 20 亿+消费者账户、1.5 亿商户、220+ 市场的网络效应

**关键冲突**：
- 当交易主体从"人"变成"人授权的智能体"，信任模型需要重建
- AMP 能否成为 Agent 时代的"HTTP"？还是会被超级 App 的围墙花园取代？

**数据支撑**：
- 蚂蚁连接 40+ 家全球最大移动支付钱包、220+ 市场、300+ 支付方式

**关键引用**：
> 朱民（前中国人民银行副行长）："当数千个智能体共存于同一个支付系统时，可能引发'海啸'级别的系统性风险。"

### 第二章：场景层 — 微信 AI 生态

> 关键词：Agent 去哪用？超级 App 如何成为"意图入口"？

- 嘟嘟巴士首批接入：一句话完成线路推荐、班次匹配、购票（官方数据：50+ 城市、10,000+ 条线路、2,000+ 企业客户）
- 龙虾出行全面接入：覆盖打车、机票、酒店、高铁、城际巴士，"0 佣金"订阅制
- 小红书 RED Skill：将 AI Skill 与社区深度融合，"内容即分发"（30万创作者，6亿+曝光）

**关键冲突**：

- 超级 App 的封闭生态 vs 开放协议：AMP 想做的事，微信会不会自己做？

**数据支撑**：

**关键事件时间线**：

### 第三章：治理层 — 国家政策

> 关键词：Agent 谁来管？安全与创新的边界在哪？

**核心叙事**：
- 2026 年 5 月 8 日：国家网信办等三部委联合发布《智能体规范应用与创新发展实施意见》
- 四大原则：安全可控、规范有序、创新驱动、应用牵引
- 关键机制：分类分级治理（高风险备案检测召回，低风险合规自测）
- 标准体系：智能体互联协议（AIP，GB/Z 185.1-185.7-2026）、智能体注册平台、数字身份管理（AIC）
- 19 个典型应用场景：科学研究（2）、产业发展（5）、提振消费（3）、民生福祉（4）、社会治理（5）
- 2027年目标：智能体应用普及率超 70%

**关键冲突**：
- 监管的"安全底线"与创新的"发展空间"如何平衡？
- 中国率先构建治理框架，是否意味着在 Agent 赛道抢占先机？

**权威引用**：
- 薛澜（清华大学）："人工智能的风险性质已经从内容风险向行为风险扩散，且影响范围更广，传导速度更快，责任主体更加模糊。"
- 钟新龙（赛迪研究院）："率先构建起覆盖智能体全生命周期的发展和治理框架，为产业发展划定了'安全底线'与'创新空间'的双重边界。"
- 邬贺铨（中国工程院院士）："标志着智能体从技术探索迈向规范发展的关键一步。"

**AIP 协议亮点**：
- 50+ 家企业试点（92% 来自国家人工智能创新应用先导区）
- 首批试点：浪潮、华为、清华大学、中国电信等 28 家单位
- 开源社区：https://aip.openatom.tech/

> 关键词：谁能成为 Agent 的"第一入口"？OS 层 vs 超级 App 层

**核心叙事**：

**关键冲突**：
- OS 层（华为/小米/OPPO）vs 超级 App 层（微信）：两种不同的 Agent 分发哲学
- 手机厂商控制硬件和系统调用，微信控制社交关系和用户时长

> 小米官方：Xiaomi miclaw "以系统应用身份运行，拥有系统核心应用能力"，"把手机的系统能力封装成了 50+ 系统能力和生态服务"

### 终章：四层的交汇与冲突

**交汇点**：
- 蚂蚁 AMP = 支付层 + 信任层（KYA 认证与分类分级治理呼应）
- 微信 AI = 场景层 + 分发层（19 个典型场景中的消费/出行/民生）
- 国家政策 = 治理层 + 标准层（为支付和场景提供合规框架）
- 手机厂商 = 终端层 + 入口层（OS 级 Agent 能力 + 硬件绑定）

**冲突点**：
- 谁来主导 Agent 商业的标准？企业（蚂蚁/腾讯）vs 国家（AIP）vs 手机厂商（OS）
- 开放协议（AMP/AIP）vs 封闭生态（超级 App 围墙花园）的路线之争
- 采用鸿沟：88% 组织在使用 AI，但仅 ≤10% 在任何单一职能中规模化部署 Agent

**Gartner 2026 预测**：
- AI Agent 软件支出：2026 年 $2065 亿，2027 年 $3763 亿
- 40% 企业应用将嵌入 AI Agent（2025 年 <5%）
- 但 >40% 的 Agentic AI 项目将在 2027 年前被取消（成本高、价值不清、风险失控）

## 2026-06-23 更新：微信Agent「小微」正式灰度上线

### 事件

2026年6月20-21日，微信原生AI助手「小微」在多个平台同时开始灰度测试——消息列表左上角新增绿色眼睛机器人图标，标注"测试版"字样。

腾讯客服确认：小微是微信团队正在灰度测试的原生AI助手，支持通过文字或语音对话操作微信原生功能，涵盖调整设置、发送消息、点外卖、生成图片等场景，正式上线时间待定。

### 四层能力模型

从被测用户反馈来看，小微的能力分为四个层次：

1. **基础执行**：语音/文字指令操控微信原生功能——发消息、拨电话、调整设置、设置提醒，一句话完成。
2. **生态调用**：识别意图后自动唤起对应小程序，在微信内完成挂号、购物、叫车等全流程。美团、滴滴、携程、京东等首批接入。微信支付为此推出"AI专属卡"——设好额度后AI在授权范围内可自主完成交易（关键步骤需用户确认）。
3. **内容理解**：发PDF或文件，输出结构化总结并展示完整思考过程。
4. **生成小工具**：用日常语言描述需求即可生成小程序雏形，支持多轮对话调整（现阶段仅限创建者本人使用）。

### 模型架构

混合路线：主模型为微信自研MoE架构WeLM（中文理解+微信生态优化），复杂推理任务调用DeepSeek兜底。

### 两张王牌

**① 小程序生态全链路闭环**：用户不再需要记住服务在哪个小程序里，只需表达需求。微信从通讯工具演变为用自然语言操作的"AI操作系统"。

**② 社交层深度打通**：小微可扫描好友圈的内容趣味（"好友最近都在点赞什么视频"）、复盘聊天对话（梳理主线、分类汇总讨论话题）、以第三人称视角分析思维盲区——这是所有第三方AI无法复制的能力。

### 边界与挑战

小米的产品哲学边界：AI是"辅助但不代劳"——可以帮你写朋友圈文案，但最终发布必须由用户完成。一旦AI规模化"代劳"真实社交行为，熟人关系链这块微信核心护城河将面临内部瓦解风险。

隐私压力：小微对聊天数据的调用能力远超外界预期，数据安全机制的透明度与合规性将长期处于放大镜下。

### 竞争格局

就在小微灰度测试前不到一周（6月16日），支付宝AI智能体"阿宝"开启内测。两款国民级应用在几天内相继亮出AI底牌——超级App AI入口争夺战正式开幕。

### 对本文的意义

微信小微的上线将本文讨论的"Agent商业基础设施"从概念推入现实。第二章（场景层—微信AI生态）的预测正在兑现：微信从"元宝失宠"到"小微接棒"，从开发者接入指引到产品正式亮相，节奏紧凑而清晰。小微的出现也印证了终章的核心冲突——**当微信自己下场做Agent入口，AMP等开放协议还剩下多少空间？**
