# DeepSeek 造芯：当软件优化打到极限，下一步只能动硬件

## 热度背景

2026 年 7 月 7 日，路透社独家报道：DeepSeek 被曝自研 AI 推理芯片，项目已秘密推进约一年。三位知情人士透露，DeepSeek 目前已完成架构设计，正在接触芯片设计公司、代工厂和存储供应商，同时低调招聘芯片工程师（未公开发布职位）。36氪在同日发布深度追踪，量子位、InfoQ 同步跟进，话题登上微博热搜。

这不是孤立事件。此前一个月，OpenAI 发布首款自研推理芯片 Jalapeño（与 Broadcom 合作，9 个月 tape-out），Anthropic 被曝与三星洽谈 2nm 定制芯片。AI Labs 集体把手伸向芯片层——但 DeepSeek 的逻辑跟 OpenAI 和 Anthropic 不一样。

OpenAI 和 Anthropic 造芯，最直接的驱动是账本——推理成本吃掉大半营收，自己做芯片能甩掉 NVIDIA 的中间商利润。DeepSeek 不是。它的动机更深一层：**软件层能榨出来的效率已经被榨干了，剩下的只能向硅片要**。

## 为什么是"必然"而非"选择"

DeepSeek 在软件效率上的记录是这个判断的底气：

- **V3（2024.12）**：671B 参数，2,048 块 H800 训练，总成本 **$557 万**——同规格模型训练成本通常是它的 10 倍以上。
- **V4（2026.04）**：1.6T 参数，每次只激活 49B（3%）。V4-Pro 输出定价 **$0.87/M token**——GPT-5.5 的 1/34，Claude Opus 4.7 的 1/29。5 月宣布 75% 永久降价。

在别人的硬件上，DeepSeek 已经把软件的账算到了骨头里。

但有三道天花板，软件够不着：

**1. MoE 稀疏激活在 GPU 上的结构性浪费。** DeepSeek 的 MoE 架构每次只激活 3% 的参数，剩余 97% 的权重沉睡。通用 GPU 的 tensor core 为密集矩阵乘法设计，MoE 的稀疏路由、expert gather/scatter 在 GPU 上有天然的调度和带宽浪费。自研芯片可以在硅片层面定制稀疏计算的数据通路——跳过零值，直连激活分支。

**2. MLA 注意力的算子转换损耗。** DeepSeek 自研的 Multi-head Latent Attention 基于低秩矩阵分解，用更小的 latent 空间表达注意力。但 GPU 跑 MLA 走的仍是标准 attention 的算子管线（QKV 投影 → 多头切分 → softmax → 加权），中间有矩阵形状转换和精度损失。自研芯片可以原生支持 MLA 的矩阵形态——不再"翻译"，直接计算。

**3. 百万 token 上下文的 KV Cache 内存墙。** V4 支持 1M token 上下文，KV cache 的容量和带宽是硬瓶颈。通用 HBM 的带宽分配是为各类 AI 负载的平均模式设计的，不是为 DeepSeek 的 KV cache 访问模式定制的。自研芯片可以按需分配片上 SRAM 和 HBM 带宽——把最贵的资源用在最疼的地方。

这三道天花板有一个共同特征：**不是算力不够，是架构不匹配**。GPU 是通用方案，DeepSeek 的模型是极端特化方案。通用的总能跑，但中间的适配成本——调度开销、算子转换、带宽浪费——正是软件优化打完最后一滴油之后，剩下的"硅片级摩擦"。

## 行业背景帧（不展开）

自研推理芯片正在成为 AI Labs 的集体转向：Google TPU 8i 首次拆分出推理专用芯片，Meta MTIA 系列全线 RISC-V、每半年迭代一代，AWS Trainium 从内置走向外售。OpenAI 的 Jalapeño 在台积电 3nm 上用 9 个月完成了 tape-out——AI 辅助芯片设计正在缩短从决定到硅片的距离。

但 DeepSeek 的特殊之处在于：它不像 OpenAI 有台积电先进制程，不像 Anthropic 有三星 2nm GAA 的备选路线。美国出口管制将它锁定在中芯国际 7nm 这一条路上。

## 造芯的"不可能三角"：团队、制程、经验

DeepSeek 造芯的威胁不在账面上——$74 亿的融资弹药充足。真正的风险在执行层：

**团队**：DeepSeek 约 150 人（同期 OpenAI ~5,000 人），创始人梁文锋的底色是量化金融 + 电子工程，不是芯片出身。芯片团队是近一年悄悄从外部招聘组建的，具体领军人、规模、背景均未公开。软件工程的极限突破能力和芯片工程的可靠性积累，是两种截然不同的能力栈。

**制程**：中芯国际 N+2（7nm）是唯一可用的国内先进节点，2026 年目标月产 7 万片——但 90% 的产能已被华为昇腾、寒武纪等头部玩家锁定。DeepSeek 作为新玩家能不能排上产能，是没有答案的问题。HBM 供应同受管制，SK 海力士、三星、美光的路都走不通，只能等华为内部 HBM 或国产替代。

**经验**：一次 tape-out 数千万美元，失败重来又数千万。DeepSeek 擅长在极限约束下做工程创新——H800 的互联带宽受限就发明 DualPipe、昇腾的算子不全就自己写 CANN 适配层。但芯片工程容错率极低：软件 bug 可以热修复，硅片 bug 只能流下一版。

这三个问题能否被 DeepSeek 式的工程能力覆盖——这是整件事最有悬念的地方。

## 与华为的微妙关系

DeepSeek V4 与华为昇腾 950DT 是深度协同设计的——SemiAnalysis 的 trace 级分析确认这是 day-0 原生适配，不是事后移植。DeepSeek 目前持有约 60,000 块 NVIDIA 芯片，另有相当数量的华为昇腾。华为是 DeepSeek 当前最大的算力供应商。

但自研芯片的终点，恰恰是要替代华为硬件在自己的推理堆栈中的位置——降低对 NVIDIA **和华为**的双重依赖。既合作又竞争，这不是对立关系，是产业分工的演化必然。

## 叙事主线

DeepSeek 的造芯不是"也来做芯片"的跟风，而是它的工程方法论的必然延伸：在软件层把极限榨干之后，唯一还能挤出效率的空间在硅片层。不是要做下一个 NVIDIA，而是要建一条不依赖任何人的推理护城河——自己设计的模型跑在自己设计的芯片上，中间的每一层摩擦都被消除。

这比单纯的"降本"更深一个维度：**不是更便宜地跑同样的计算，而是把计算的形态本身重塑成模型想要的形状。**

## 创作方向建议

1. **引子**：路透社爆料——DeepSeek 秘密造芯一年，从"降本"切入但不限于降本
2. **软件效率已见顶**：硬件上跑出的三道天花板（MoE 稀疏浪费、MLA 算子损耗、KV Cache 内存墙）
3. **从软到硬的跃迁**：DeepSeek 式工程思维——不是做更便宜的 GPU，是把模型的架构写进硅片
4. **风险三板斧**：团队从零搭建、中芯 7nm 产能排队、芯片工程零容错
5. **与华为的关系**：合作伙伴还是竞争对手？既合作又竞争的产业必然
6. **收束**：DeepSeek 造芯的本质不是芯片战争，是软硬一体化的工程哲学——值得做，更难做
