## 核心背景

### 发生了什么

2026 年 6 月 27 日，DeepSeek 联合北京大学正式发布 DSpark 推理加速框架。梁文锋亲自署名论文。训练代码、模型权重、评估脚本全量开源在 GitHub DeepSpec 项目。

**核心指标**：同等吞吐量下，单用户生成速度提升 60%–85%。已在 DeepSeek-V4-Flash 和 DeepSeek-V4-Pro 预览版引擎中部署。

这是 DeepSeek 完成首轮 500 亿融资后的第一个重大技术发布。

### 技术看点

DSpark 解决了推测解码（speculative decoding）在实际部署中的核心矛盾：

- **半自回归架构**：并行主干网络（基于 DFlash 改进）一次性产出全部候选位置的隐藏状态，再由轻量级顺序模块（马尔可夫头 / RNN 头）逐 token 注入前缀依赖——兼顾并行效率与接受率
- **置信度调度验证机制**：模型在每个候选位置输出置信度分数，硬件感知前缀调度器动态决定每个请求验证多长的候选前缀，优先将 GPU 资源分配给全局存活概率最高的 token
- **生产环境实测**：V4-Flash 引擎在 80 token/s SLA 下吞吐量提升 51%；V4-Pro 引擎在 35 token/s SLA 下吞吐量提升 52%

### 战略含义

DeepSeek 融资后选择开源推理方案（而非封闭商业化），这个选择本身就是信号：

1. **开源护城河**：推理加速框架全量开源 → 吸引社区贡献 → 生态绑定 → 即使别人追上模型能力，推理层仍在 DeepSeek 技术栈上
2. **API 成本杀手**：推理加速 = 同样硬件跑更多请求 = 更低 API 价格。DeepSeek 本就以低价著称，DSpark 进一步拉大成本差距
3. **"中国 Infra 首次定义全球标准"**：DSpark 同时开源了 DFlash 和 Eagle3 两种基线方法的训练代码，正在成为推测解码领域的标准基准框架
