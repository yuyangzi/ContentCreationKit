# Vibe Coding：昙花一现还是真改变编程范式

## 热度背景

2026年7月，知乎"你觉得Vibe Coding是昙花一现还是真的在改变编程这件事？"话题获得113万热度，引发大量技术社区讨论。

"Vibe Coding"指的是开发者通过与AI对话（而非手写代码）来完成软件开发的编程范式。支持者认为它降低了编程门槛，让产品经理甚至非技术人员都能"写代码"；反对者则指出Vibe Coding产出的代码难以维护、缺乏架构设计，更像是"AI时代的低代码"。

在掘金、GitHub等开发者社区，围绕Vibe Coding的讨论已从"这是什么"进化到"什么时候用、什么时候不用"的阶段。有开发者声称靠Vibe Coding一天上线MVP，也有人在维护Vibe Coding产物时崩溃。

其背后的核心张力是：当AI能写出90%的代码时，剩下10%的判断力和架构能力，才真正定义了程序员的价值。

## 创作方向建议

1. **现象分析**：Vibe Coding到底是什么？它和低代码、无代码的本质区别在哪？
2. **能力边界**：Vibe Coding擅长什么、不擅长什么？从具体案例出发划清边界
3. **深层影响**：如果Vibe Coding成为主流，程序员的技能栈需要如何重构？
4. **叙事角度**：从"两种程序员的分化"展开——会用AI的和不会用AI的，差距正在以指数级拉大
