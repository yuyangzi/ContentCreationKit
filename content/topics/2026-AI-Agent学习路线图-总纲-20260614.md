## 学习路径总览

### 正确的学习顺序（2026 版）

```
阶段一：地基 —— 理解 Agent 的运行时心脏（1-2 周）
  ↓
阶段二：手脚 —— RAG + LangChain + 第一个单 Agent（1-2 周）
  ↓
阶段三：协作 —— LangGraph + MCP 协议 + 多 Agent（2-3 周）
  ↓
阶段四：实战 —— SWE-Bench + 企业部署 + 性能优化（持续）
```

### 常见误区

**❌ 错序**：先学框架 → 搭 demo → 遇到真实场景就崩
**✅ 正序**：先搞懂原理层面的坑 → 再学框架怎么填坑 → 最后用框架加速

## 四阶段详解（各一篇深扎教程）

### 阶段一：地基——理解 Agent 的运行时心脏

**为什么不能跳过**：不理解 Function Calling 的底层机制（tool_choice → JSON Schema → finish_reason 三段式流水线），直接用 LangChain 的 `@tool` 装饰器就是在黑盒操作。

**核心模块**：
- LLM 基础（20%）：推理 vs 训练、为什么 LLM 输出不可靠
- 上下文工程（25%）：System Prompt 架构、工具描述工程、对话历史管理策略
- Function Calling + Agent Loop（40%）：Tool Schema 设计原则、观察→规划→执行→观察循环
- Harness Engineering 驾驭工程（15%）：约束边界设计、Human-in-the-loop 触发条件、安全护栏

**对应深扎教程**：`2026-AI-Agent学习路线图-阶段一-运行时心脏-20260614.md`

---

### 阶段二：手脚——RAG + LangChain + 第一个单 Agent

**为什么不能跳过**：Agent 的"知识手脚"——没有 RAG，Agent 只能靠训练数据回答问题；没有 LangChain 基础，无法理解社区 90% 的教程和面试题。

**核心模块**：
- RAG（检索增强生成）：Chunking 策略、向量检索 vs 关键词检索、Hybrid Search
- LangChain 六大模块：Models、Prompts、Chains、Memory、Indexes、Agents
- 构建第一个单 Agent：天气查询 + 数据库检索 + API 调用

**对应深扎教程**：待创建

---

### 阶段三：协作——LangGraph + MCP + 多 Agent

**为什么不能跳过**：2026 年 Agent 已经从"单打独斗"进入"团队协作"。MCP 是 Agent 的工具标准化协议（被 OpenAI/Google/微软/Cursor 全面采纳），LangGraph 是复杂状态管理的事实方案。

**核心模块**：
- LangGraph：图结构编排、状态管理、Human-in-the-loop
- MCP 协议：Server/Client 架构、Tool/Resource/Prompt 三元组、2026 年生态现状
- 多 Agent 协作：CrewAI 入门、Agent 间通信、避免循环依赖

**对应深扎教程**：待创建

### 阶段四：实战——SWE-Bench + 企业级部署 + 性能优化

**为什么不能跳过**：教程能跑 ≠ 生产能跑。真实工程环境面临流式输出、错误重试、安全防护、成本控制等问题。

**核心模块**：
