# Claude Code隐写术标记中国用户：技术拆解与信任危机

> 热度：⭐⭐⭐⭐⭐

## 主题背景

2026年6月30日，Reddit用户**LegitMichel777**在r/ClaudeAI发帖曝光：Claude Code CLI工具内置了一套隐蔽的检测机制，通过**文本隐写术**在系统提示词中对中国用户打标记——修改"Today's date is..."的日期分隔符（`-` vs `/`）和撇号Unicode编码（标准`U+0027` vs 特殊字符`U+2019`/`U+02BC`/`U+02B9`），对不同身份的用户（中国普通用户 / 中国AI实验室 / 两者皆是 / 皆否）打上4种不同标签。开发者Thereallo随后独立确认并发布完整逆向工程代码，冲上Hacker News榜首。

值得一提的是，这并非Anthropic第一次被发现静默操作用户系统。同年4月，隐私顾问**Alexander Hanff**已发现Claude Desktop客户端向7个Chromium浏览器静默注入Native Messaging配置文件，预授权浏览器扩展绕过沙箱通信。

技术细节：Anthropic没有修改数据包结构，而是在系统提示词的**文本格式**上做手脚——视觉上完全一样的字符，底层Unicode编码不同。域名黑名单（147条，包含百度/阿里/字节/月之暗面/MiniMax等）通过Base64+XOR(key=91)混淆存储在二进制中。后端扫描这些编码即可瞬间识别用户身份，即使通过代理也无法绕过（检测的是系统时区而非IP）。

**背景因果链**：隐写术并非凭空出现。6月10日，Anthropic向美国参议院银行委员会指控阿里巴巴Qwen实验室使用约2.5万个虚假账户、进行2,880万次对话来蒸馏Claude模型——被称为"史上最大规模AI蒸馏攻击"。从蒸馏指控到隐写标记，再到阿里禁用Claude、工信部定性"后门"，这是一条完整的连锁反应链。

行为被安全社区斥为"间谍软件"和"后门"——拥有shell权限和文件系统访问权的工具，未经披露植入隐蔽检测逻辑，用XOR混淆逃避代码审计。此事在知乎、微博等平台引发广泛讨论，微博话题"Claude偷偷给中国用户打水印"登上热搜。

## 创作方向建议

### 角度一：技术解剖——隐写术如何工作

从Unicode编码差异到提示词注入机制，一步步拆解这套"水印系统"的技术原理。为什么视觉上完全一样的字符可以承载信息？文本隐写术在AI安全领域还有哪些应用（正面和负面）？

### 角度二：这到底算不算"间谍软件"？

从法律和技术双重视角讨论：静默注入浏览器配置文件是否构成"未经授权篡改第三方软件"？用隐写术隐藏检测行为是否构成"欺骗"？和传统间谍软件的定义有何异同？

### 角度三：AI公司的"地理围栏"困境 【背景支撑，不展开为主线】

从出口管制到隐写术检测，AI公司正越来越多地采用技术手段来执行地理限制。但这条路的尽头是什么——每家大模型公司都内置一套"用户画像系统"？这对AI行业的信任基础意味着什么？

## 2026-07-09 审核修正

### 事实纠正

- **发现者归属**：隐写术检测由LegitMichel777/Thereallo发现（6月30日Reddit），非Alexander Hanff。Hanff发现的是Claude Desktop浏览器注入（4月18日）——两个独立事件，已串联为"惯犯"叙事线索
- **补充蒸馏指控背景**：Anthropic 6月10日向参议院指控阿里Qwen用2.5万假账户、2880万次对话蒸馏Claude——这是隐写术的直接动机

### 创作方向确认（grill 五轮）

- **主线**：角度一（技术解剖）+ 角度二（间谍软件定性）双线交织
- **技术深度**：写到第二层（四态标记系统 + XOR混淆），不展开第三层（ANTI_DISTILLATION_CC、追踪像素）
- **文章终点**：收在信任危机与反思——"拥有shell权限和文件系统访问权的工具，用隐写术偷偷打标，对AI工具行业信任基础意味着什么"
- **蒸馏背景**：2-3段快速交代因果链，不做技术展开
- **角度三**：作为背景支撑，不展开为主线
