# 不用向量数据库，不用 embedding——Claude Code 记忆系统的设计哲学

## 主题定位

- **领域**：AI Agent 大模型
- **类型**：设计哲学 / 架构思辨
- **目标读者**：AI Agent 开发者、对 Agent 架构有兴趣的工程师
- **确认时间**：2026-06-13

---

## 热度背景

### 事件概述

Claude Code 是 Anthropic 推出的 AI 编程 Agent，具备一项核心能力——**跨会话记忆**：能记住项目架构、用户代码风格偏好、常见 Bug 修复方案等。2026 年 3 月 31 日，Anthropic 因 npm 打包失误意外泄露了约 51.2 万行 TypeScript 源码，记忆系统的完整架构首次被外界完整解析。

结果出乎意料：**Claude Code 没用向量数据库、没用 embedding、没用任何复杂存储引擎。它用的是磁盘上的 markdown 文件。**

### 热度数据

- **掘金**：得物技术《让 Claude Code 拥有自我进化和记忆系统》306 热度
- **SegmentFault**：得物技术文章头条推荐
- **云栈社区**：《Claude Code 四层记忆系统源码解剖》《记忆机制源码解析》等多篇深度分析
- **GitHub**：大量开发者基于泄露源码进行二次分析

### 核心发现

Claude Code 的记忆系统有四层架构：

| 层级 | 名称 | 作用域 | 本质 |
|------|------|--------|------|
| 第一层 | 会话记忆 | 当前对话 | 对话历史的滑动窗口 + 摘要压缩 |
| 第二层 | 持久记忆（Auto Memory） | 项目级 | markdown 文件，四种固定类型（user/feedback/project/reference） |
| 第三层 | CLAUDE.md | 项目/用户/组织级 | 开发者手动编写的持久指令 |
| 第四层 | 团队记忆（Team Memory） | 团队级 | OAuth 服务端同步，共享上下文 |

设计亮点：
- **不用向量数据库**：结构化文件 + LLM（Sonnet）做"选择题"判断相关性，而非 embedding "搜索题"
- **只得四种类型**：user / feedback / project / reference，强制分类约束信息熵
- **索引常驻 + 内容按需**：MEMORY.md 始终加载（最多 200 行/25KB），具体记忆按需检索（最多 5 条）
- **非侵入式提取**：每轮对话结束后 fork 子 Agent 提取记忆，不阻塞主对话
- **老化提醒**：2 天前的记忆主动标记 stale，逼模型验证而非盲信

## 创作方向建议

### 核心叙事线

1. **引子（10%）**：源码泄露揭开的真相——最强 AI 编程 Agent 的记忆，存的是 markdown 文件。这不只是"简单"，是经过深思熟虑的"做减法"
2. **为什么不是向量数据库？（20%）**：
   - 向量检索的诱惑：语义搜索、自动关联、"看起来很 AI"
   - 真实代价：向量化开销、索引维护、嵌入模型依赖、隐私泄露风险
   - 关键洞察：Agent 记忆不是"搜文档"，是"做判断"——判断一条记忆是否相关，需要推理能力，不是余弦相似度
3. **"四种类型"的设计智慧（20%）**：
   - 为什么是四种？分类即压缩——预先规定信息结构，比事后用 embedding 聚类高效得多
   - 每种类型的职责边界：user（偏好）、feedback（纠正）、project（架构）、reference（命令/文档）
   - 类比：SQL 的强 schema vs NoSQL 的文档自由——不是"复古"，是"工程取舍"
4. **索引常驻 + 内容按需——分层加载的精妙（15%）**：
   - MEMORY.md 是索引不是记忆——防止单文件膨胀，支持截断而不丢失核心信息
   - 用 Sonnet（不是 Haiku）做 5 选 1：因为判断"哪条记忆和当前问题相关"需要推理能力
   - 老化机制：stale 标记不是禁用，是"提醒模型重新验证"
5. **fork 子 Agent——非侵入式提取的巧思（15%）**：
   - 每轮对话结束后悄悄 fork，不阻塞主对话
   - 互斥检查：主 Agent 已写入 → 跳过自动提取，防止覆盖用户意图
   - 默默学，不打扰
6. **结尾（20%）——设计哲学的三条原则**：
   - **结构化优于自由文本**：用类型约束换检索精度
   - **廉价模型做选择题**：不去"理解"所有记忆，只是"选出相关的"
   - **时间感知 + 主动验证**：记忆会过时，系统应该自己知道
   - 核心命题：AI 记忆系统的设计不是"加更多技术"，是"减到不能再减"
