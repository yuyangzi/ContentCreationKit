# Claude Code 80%提示词删了：AI降本新范式

## 热度背景

2026年7月，Anthropic用Fable 5重写了Claude Code，将提示词（prompt）删掉了80%，效果反而更好。

据InfoQ报道，Claude Code的新版本通过模型能力的提升，大幅减少了工程化的提示词设计。过去一年，行业在"提示工程"（prompt engineering）上投入了大量人力和算力——精心设计prompt模板、A/B测试措辞、优化few-shot示例——但现在发现，模型变强后，这些投入的价值直线下降。

同期，36Kr报道"Claude工程师终于交出Fable 5焚诀，教你打破和模型之间的信息差"，进一步揭示了模型迭代速度远超提示工程迭代速度的现实。

这一事件标志着AI行业一个重要拐点：当模型能力跨越某个阈值后，"用prompt补模型短板"的策略开始失效，取而代之的是"让模型自己搞定"——这同时意味着依赖提示工程构建的商业模式和产品壁垒，可能瞬间崩塌。

## 创作方向建议

1. **技术分析**：到底是什么让提示词可以删掉80%？Fable 5和Claude Code的协同进化
2. **行业影响**：提示工程作为一个职业/技能还有什么价值？那些靠"prompt模板"建立的SaaS还有护城河吗？
3. **降本逻辑**：提示词越短→token消耗越少→成本越低。AI降本不是靠压价，而是靠"更聪明的模型"
4. **叙事角度**：从"提示工程的消亡"展开——一个诞生不到两年就被技术演进"碾碎"的工种，折射出AI行业迭代的残酷性
