# Google Chrome 提出 WebMCP 标准提案：原生 Web 智能体协议

## 主题名称

**Google Chrome 提出 WebMCP 标准提案：原生 Web 智能体协议**

## 热度背景分析

### 背景

Chrome 已经在 6 月启动 WebMCP 标准提案的 Origin Trial 阶段。WebMCP 是浏览器原生协议，允许 AI 智能体以编程方式操作网页，相当于给浏览器内置了 AI 智能体操作接口，不用再靠 Playwright 等外部工具做网页自动化。可能成为 AI 智能体访问网页的标准接口，就像 MCP（Model Context Protocol）用于文件/工具访问一样。

### 热度趋势

- 发布后在技术社区引发了广泛讨论
- 标准还处在早期试验阶段，关注度持续上升
- 很多 AI 智能体开发者已经在测试可用性

### 受众核心关注点

1. **技术价值**：WebMCP 解决了现有 AI 智能体操作网页的哪些痛点？
2. **行业影响**：如果成为标准，对现有网页自动化工具、RPA 行业会有什么冲击？
3. **未来展望**：AI 原生浏览器会变成什么样？这会是浏览器 AI 时代第一个基础性标准吗？
4. **兼容性**：其他浏览器厂商（Firefox、Safari）会跟进支持吗？

## 创作方向建议

### 推荐方向一（技术深度）：**WebMCP 是什么？为什么说它可能是 AI 智能体的基础设施？**

切入点：
- 现有方案（Playwright/Puppeteer）的痛点：模拟点击、不稳定、性能差
- WebMCP 设计思路：原生协议、直接暴露 DOM API 给智能体
- 和 MCP（Model Context Protocol）关系是什么？互补还是竞争？
- 对 AI 智能体开发意味着什么？降低了多少门槛？

适合人群：AI 智能体开发者，技术深度向。

### 推荐方向二（产业影响）：**当浏览器原生支持 AI 智能体 — RPA 和网页自动化行业会被颠覆吗？**

切入点：
- 如果 WebMCP 普及，UiPath、Automation Anywhere 这些 RPA 厂商怎么办？
- 对现有爬虫行业有什么影响？
- 中小开发者能更容易开发网页 AI 自动化了吗？
- 这会催生更多 AI 自动办公应用吗？

适合人群：关注产业变化、自动化领域读者。
