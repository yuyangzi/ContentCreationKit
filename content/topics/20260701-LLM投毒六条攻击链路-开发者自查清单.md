# 从Pip安装到模型训练：LLM投毒的6条攻击链路

> 热度：⭐⭐⭐

## 主题背景

2026年，AI投毒已不再是学术理论——Lakera、Checkmarx、Knostic等多家安全公司发布了实战报告，共同描绘出一张完整的LLM攻击面地图。

六条攻击链路：
1. **预训练数据**：污染训练语料库（维基百科、Common Crawl等），植入长期后门
2. **微调数据**：在企业微调过程中注入少量有毒样本，精确控制模型在特定场景下的行为
3. **RAG知识库**：污染向量数据库中的文档，使AI在检索增强生成时输出恶意内容
4. **Agent工具定义**：篡改Agent的工具描述（function calling schema），让Agent执行非预期的操作
5. **pip/npm依赖**：在AI框架依赖的Python包中注入恶意代码（LiteLLM事件即属此类）
6. **系统提示**：通过第三方prompt市场投放条件性后门（PARASITE攻击）

每一条链路都有真实案例，且攻击成本和防御难度差异极大。这是2026年AI安全领域最重要的全景式图景之一。

"你的AI项目正在被攻击的6个入口（附自查清单）"——为开发者提供一份可直接使用的安全检查表，每条链路配一个具体攻击案例和防御建议。

### 角度二：攻击面地图
以一张可视化图表为核心，展示LLM开发全流程中每一层可能受到的攻击。从数据采集到部署运行，每个阶段有哪些威胁，对应什么防御工具。

用LiteLLM事件作为钩子引入，然后展开讲攻击面全景——LiteLLM只是6条链路中的一条，其他5条同样危险且已在实战中被利用。
