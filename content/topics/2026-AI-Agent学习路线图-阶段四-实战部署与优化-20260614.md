### 模块一：SWE-Bench Verified 实战（35%）

**SWE-Bench Verified 是 2026 年 Agent 工程能力的黄金标准——它的 500 个真实 GitHub Issue 比任何教程都更接近"真实工作"。**

| 知识点 | 核心内容 |
|--------|----------|
| SWE-Bench 是什么 | 500 个真实 GitHub Issue → Agent 自动修 Bug → 自动评测 |
| SWE-Bench Verified vs 原版 | 去除了含 Hint 的 Issue，纯工程能力评测 |
| Code Agent 架构 | 代码搜索 → 文件定位 → 代码编辑 → 测试验证 的四步闭环 |
| 工具链 | 文件搜索（grep/ast-grep）、代码编辑（精确替换）、测试执行 |
| SWE-Agent / OpenHands 简介 | 开源 Code Agent 框架的架构解析 |
| 成功率提升策略 | 更好的搜索策略、更精确的编辑工具、多轮自我验证 |

**实操项目**：用阶段一至三学到的技能，在 SWE-Bench Lite 上跑一个简单 Issue，理解 Code Agent 的完整闭环。

---

### 模块二：企业级部署（35%）

**Demo Agent → 生产 Agent 的 5 个必过关卡。**

| 关卡 | 问题 | 解决方案 |
|------|------|----------|
| 流式输出 | 用户等不了 Agent 完整执行完才看到结果 | SSE（Server-Sent Events）、逐 token 流式返回 |
| 错误重试 | LLM 调用失败、工具超时、网络抖动 | 指数退避重试 + 熔断器 + 最终降级策略 |
| 安全防护 | Prompt Injection、敏感信息泄露、越权操作 | 输入过滤、输出过滤、沙箱执行、权限最小化 |
| 对话记忆管理 | 长对话导致 token 超限、上下文污染 | 滑动窗口 + 摘要压缩 + 选择性保留（阶段一上下文工程的工程化落地） |
| 可观测性 | Agent 在执行中——但你看不到它在干什么 | LangSmith / LangFuse 追踪、日志结构化、关键指标监控 |

**实操项目**：将阶段三的多 Agent 系统加上流式输出 + 错误重试 + LangFuse 追踪。

### 模块三：性能优化三维度（30%）

**Agent 的性能不是单一的"跑得快"——精度、延迟、成本是相互制约的三角。**

```
           精度 ↑
           /\
          /  \
         /    \
        /  优化 \
       /  目标区  \
      /____________\
   延迟 ←-----------→ 成本
```

| 维度 | 优化手段 | 权衡 |
|------|----------|------|
| **精度** | 更好的 System Prompt、Few-shot 示例、Rerank 精排、Self-Reflection | 增加延迟和 token 消耗 |
| **延迟** | 更小的模型（Flash 系列）、并行工具调用、流式输出、缓存 | 可能降低精度 |
| **成本** | 更便宜的模型、Prompt 压缩、缓存命中、限制上下文长度 | 可能降低精度和增加延迟 |

| 优化策略 | 适用场景 | 效果 |
|----------|----------|------|
| Prompt Caching | 重复的 System Prompt + 工具定义 | 减少 50-90% 首 token 延迟 |
| 并行 Tool Calling | 多个独立的工具调用 | 延迟从串行变并行 |
| 模型降级路由 | 简单任务用小模型，复杂任务用大模型 | 大幅降低平均成本 |
| 上下文压缩 | 长对话场景 | 降低 token 消耗，可能损失信息 |

**实操项目**：对比同一个 Agent 在 DeepSeek V4-Flash vs DeepSeek V4-Pro 上的精度/延迟/成本差异。
