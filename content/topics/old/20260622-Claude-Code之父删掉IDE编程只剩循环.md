# Claude Code 之父：删掉 IDE，编程只剩循环

## 热度背景

- **热度信号**: ⭐⭐⭐⭐
- **类型标签**: [范式]（编程方法论层面的深层转换，非短期事件）
- **发布时间**: 2026-06-22
- **数据来源**: 36氪（8446 阅读）、开发者社区热议

Claude Code 创建者（核心开发负责人）公开表态自己已彻底删除 IDE，不再编写提示词（prompt），编程只剩下"写循环"——让 AI 迭代执行代码，人类只负责方向判断和结果验收。这是对"什么是编程"这一根本问题的重新定义。

## 核心关注点

1. **编程的范式转移**：从"写代码"到"指挥 AI 写代码"——当人类不再逐行编写逻辑，程序员的角色如何重新定义？
2. **"只写循环"的具体含义**：让 AI 反复执行-检查-修正的闭环——这是通用方法还是仅适用于特定场景？
3. **与现有技术的关系**：这是对 Vibe Coding 的极端版本，还是有本质区别？Vibe Coding 强调"让 AI 写一切"，而"循环模式"强调人类仍控制验证和方向
4. **IDE 的未来**：如果顶尖 AI 编程工具的创造者自己都不再用 IDE，JetBrains 和 VS Code 真的还有未来吗？
5. **教育启示**：如果编程的未来是"循环思维"而非"代码能力"，CS 教育应该教什么？

## 创作方向建议

- **思想实验向**：拆解"只写循环"的工作流——用具体的编程任务（如构建一个 API 服务）一步步展示这个模式怎么操作，优缺点是什么
- **对比分析向**：Vibe Coding → Loop Engineering → 纯方向判断——编程范式演变的三个阶段
- **批判性角度**：这种极端观点是否只是一种"幸存者偏差"？Claude Code 的作者是 Claude Code 的重度用户，对自己产品的极致信仰是否适用于普通开发者？
