# 审核草稿

## 目标

对文章草稿进行 StyleRule 五条规则逐项检查、数据真实性核对和逻辑连贯性审核。

StyleRule 完整版见 `StyleRule.md`，以下清单是执行时的逐项检查点。

## 前置条件

已完成 `/create-draft`，`content/draft/` 目录已有草稿文件

## 输入

- `content/draft/` 目录下的草稿文件
- `StyleRule.md`——编辑风格规则（完整版）

## 步骤

### 1. 作者隐身 (StyleRule §1)

**核心**：不让读者看到作者在"组织内容"。删除展示写作过程、替读者提问、预告剧情的句子。"我"是叙事底色，不是前台焦点。

逐项检查：
- [ ] **元叙述**：没有"写到这里/我注意到/讲到这里就绕不开"——直接推进，不展示写作行为
- [ ] **剧情预告**：没有"接下来你会看到/更精彩的还在后面"——信任叙事自然展开
- [ ] **认知反转展示**：没有"一开始我也是这么想的，但查完资料发现事情不是这样"——直接给结论
- [ ] **设问修辞**：没有替读者提问再自答（"讲到这里就绕不开一个问题……答案要从XX说起"）——直接说XX
- [ ] **"有趣的是/令人意外的是"**：没有这类情绪标签——直接陈述事实
- [ ] **密集"我"**：没有连续出现的"我也/我发现/我注意到"——删到只剩必要的
- [ ] **判断标准**：删掉这句话，读者还能不能跟上叙事？能就删。

### 2. 弱化绝对 (StyleRule §2)

**核心**：对"斩钉截铁"的判断有本能反感。用"大多数/通常/倾向于"替代"全都/从来不/出奇一致"。不对未来做武断预测。

逐项检查：
- [ ] **绝对断言**：没有"全都/从来不/不管XX都不会"式斩钉截铁判断
- [ ] **未来预测**：有对未来判断时加了限定（"至少当前/目前来看"）
- [ ] **过度概括**：没有把个别案例包装为普遍规律
- [ ] **判断标准**：这句话有没有把话说死？有就加限定。

### 3. 论据收敛 (StyleRule §3)

**核心**：每个论据只直接支撑当前段落论点。论据够了就收，例子懂了就停，数据到位就止。

逐项检查：
- [ ] **程度重申**：对比已成立，没有补"整整早了X时间"类冗余
- [ ] **数据堆砌**：一组数据足以支撑判断后，没有追补更多数据"再证明一遍"
- [ ] **多例枚举**：同一论点没有用两三个并列例子反复论证——留一个
- [ ] **背景铺垫**：引用数据前没有花半段介绍公司背景——直接给数据
- [ ] **防杠声明**：没有"截至X时间"这类不影响论点的时效声明
- [ ] **非直接证据**：段落信息与当前论点不直接相关——已删除
- [ ] **来源标注**：关键数据首次出现括号附注来源，后文不重复标注
- [ ] **判断标准**：去掉这条信息，本段核心判断是否变弱？不变弱就删。

### 4. 叙事纯净 (StyleRule §4)

**核心**：三段纪律——段内论据不冗余、段级不跑题、段间不岔路、过渡不吓人。

逐项检查：
- [ ] **新闻聚合段**：没有"有人这么说、有人那么说"的综述段——这是作者的分析，不是新闻汇编
- [ ] **背景推论岔路**：主线论证不是建立在"事实如此"上，没有插入"为什么发生"的推论岔路
- [ ] **对比案例岔路**：论证A时没有"顺便提一下B"的对比岔路
- [ ] **戏剧化过渡**：没有"如果唯速度论，企业可能会栽到另一个坑里"类预警——改为"但速度还不是全部"
- [ ] **破折号打断**：偏好括号（XX）安静补充，少用破折号（——XX——）强行打断句子
- [ ] **过渡句纪律**：过渡只做方向转折（"但/然而"）或逻辑推进（"更进一步/这意味着"），不附加情绪悬念
- [ ] **判断标准**：从这段读到下一段开头，需要切换语境才能理解方向转换？这就是岔路，删。

### 5. 列表化 (StyleRule §5)

**核心**：3个及以上并列特征/原因/维度用 `1. 2. 3.` 或 `-` 列表呈现，不让读者在段落里自己找要点。

逐项检查：
- [ ] **并列结构**：3+同级并列信息已用列表，而非装在散文段落中
- [ ] **适度使用**：没有追求"全篇列表化"——只在需要一次性接收多个并列信息时才用
- [ ] **排版规范**：列表前后各加空行
- [ ] **判断标准**：这些要点是同级并列关系吗？是就改列表。

### 6. 综合检查

- [ ] 文章的数据和内容真实且未过时
- [ ] 上下文结构合理，前后逻辑正确，语句通顺
- [ ] 如存在任何问题，给出修正意见供我审核

## 输出

- 审核意见清单（在对话中呈现），按 StyleRule 五条规则逐项报告，每条附带具体例证

## 约束

**必须遵守：**

- 五条 StyleRule 必须逐条逐项检查，不能跳过或笼统带过
- 每项检查需给出具体问题位置（引用原文），不笼统说"已通过"
- 每条 StyleRule 检查后附判断结论："通过" 或 "需修改"

**禁止操作：**

- 不要直接修改草稿文件
- 不要只给笼统评价，必须指出具体问题位置

**参考：**

- `StyleRule.md` 包含各规则的更多示例和判断标准
- AGENTS.md 中 StyleRule 章节为精简版，审核时以完整版为准
