# 审核草稿

## 目标

对文章草稿进行 AI 去味检查、数据真实性核对和逻辑连贯性审核。

## 前置条件

已完成 `/create-draft`，`content/draft/` 目录已有草稿文件

## 输入

- `content/draft/` 目录下的草稿文件

## 步骤

1. **AI 去味检查**：逐项确认以下清单，每项需给出具体例证：
   - [ ] 没有"首先/其次/最后/综上所述"等模板化连接词
   - [ ] 没有过于完美的三段式结构
   - [ ] 长短句交错，不是每段都一样长
   - [ ] 有合适的情感词
   - [ ] 有适当的口语化表达
   - [ ] 有合适的括号补充
2. **个人风格适配检查**：对照以下清单逐项检查，不符合的给出修改建议：
   - [ ] **元叙述**：没有"写到这里/我注意到/一开始我也这么想但查了资料后发现"这类展示推理过程或写作行为的句子
   - [ ] **绝对断言**：没有"斩钉截铁"式判断（如"全都/从来不/不管XX都不会"），必要时加限定（至少/当前/目前来看）
   - [ ] **冗余说明**：论据点到为止——对比已成立就不补时间数字，引用已有就不补阅读过程，同件事不拆成两段说
   - [ ] **过细例子**：论点本身成立时例子可就此打住，不需要用最后一个品牌或最后一项枚举来"再确认一遍"
   - [ ] **列表化**：3 个及以上并列特征/原因/维度已用列表结构呈现，而非装在散文段落里让读者自己找
   - [ ] **第一人称**："我"不抢戏——没有"我发现/我注意到/我反复看了几遍"；没有展示"我是怎么从X变成Y的"的完整过程
   - [ ] **数据堆砌**：数据完成了逻辑支撑就收住，没有继续堆一组来"再证明一遍"
   - [ ] **设问修辞**：没有"讲到这里就绕不开一个问题"这类替读者提问再自答的过渡句
3. 检查文章的数据和内容是否真实且未过时。
4. 检查内容连贯性：上下文结构是否合理，前后逻辑是否正确，语句是否通顺。
5. 如存在任何问题，给出修正意见供我审核。

## 输出

- 审核意见清单（在对话中呈现），每项检查附带具体例证

## 约束

**必须遵守：**

- AI 去味检查清单必须逐项过，不能跳过
- 每项检查需给出具体问题位置，不笼统说"已通过"

**禁止操作：**

- 不要直接修改草稿文件
- 不要只给笼统评价，必须指出具体问题位置
