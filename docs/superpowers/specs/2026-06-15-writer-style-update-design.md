## 背景

当前 `writer-style` skill 基于旧版中文前端技术博客风格（`&emsp;&emsp;` 缩进、"什么是X?" 标题、blockquote 引用模式、粗浅总结结尾等），与 `content/article/` 下实际产出的 9 篇文章风格严重脱节。实际文章为深度科技分析/评论风格。

## 目标

以 `content/article/` 下文章的实际风格为唯一标准，重写 skill，移除不再使用的旧规则，使用中文作为主要语言。

## L1 核心规则（始终生效）

对所有文章强制执行，不区分场景。

### 1. 句子风格
- 中长句为主（2-4行），用逗号连接子句
- 允许短句制造节奏变化（"就是这么简单。"）
- 常用连接词：而、所以、但是、因为、事实上、简单来说
- 避免过度 hedging（"可能也许大概"）

### 2. 技术术语
- 保留英文原形，首次出现加中文翻译：`可观察对象（Observable）`
- 加粗仅用于核心术语，不过度使用
- 代码语言标签按实际语言（` ```python` 等），不再强制大写 J

### 3. AI 痕迹避免
- 禁止：英文过渡词、三重并列、"不仅...而且..."、em dash 过度使用
- 禁止：夸大修饰（"至关重要""革命性"）、空洞收尾、对话式 AI 回应

### 4. 语气
- 冷静、言之有物
- 可用"我"表达个人判断，但不过度
- 允许"说白了""翻译成人话"等接地气转换，适度使用

## L2 文章结构模板（唯一模式）

适用于：行业分析、技术趋势、事件评论、产品解读、深度教程。

### 开篇（Hook → Context → Thesis）
- Hook：反常识问题、冲击数字、或具体场景
- Context：2-3段背景展开，堆数据/事实/引用建立可信度
- Thesis：全文核心论点，自然过渡到正文

### 章节体系
- `## 一、[论点式标题]` 数字编号 + 冒号分隔的中文标题
- 每章一个独立论点，章间逻辑递进
- `---` 分隔大章

### 叙述节奏
- 长论证段（3-5句）→ 短强调句（1句）交替
- 数据融入论证流，不冷列
- 引用（人名、论文、报道）嵌入叙事
- "说白了""翻译成人话" 关键处做概念降维

### 代码嵌入（按需）
- 引入文字 → 代码块 → 关键行解释（不逐行注释）

### 结尾
- 自然收束，不强行升华
- 常用：信息来源标注 / 开放问题 / 回到开篇呼应

## 移除的旧规则

以下规则从 skill 中完全移除：

| 旧规则 | 原因 |
|---|---|
| `&emsp;&emsp;` 段落缩进 | 实际文章中零使用 |
| "什么是X?" 问题式标题 | 实际文章中零使用 |
| blockquote 引用 → 解释 → 代码模式 | 实际文章中极少使用 |
| "粗浅的总结" 结尾模板 | 实际文章中零使用 |
| `\`\`\`JavaScript` 强制大写 J | 实际文章按语言使用 |
| 横幅图片 + 兼容性截图结构 | 实际文章中无此结构 |
