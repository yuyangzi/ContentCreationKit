# `/find-popular-topics` 命令优化设计

**日期**：2026-06-21
**状态**：设计稿

## 背景

`/find-popular-topics` 是内容创作管线的第一步。在 3 天的使用中暴露出以下问题：

1. **搜索成本高** — 每次启动 2 个 librarian（中文+全球），工具调用重叠严重
2. **话题重复** — 同一话题（如 Claude Fable 5 下架）连续 3 天出现在不同会话中，无饱和检查机制
3. **话题分类混乱** — 时效热点和长线趋势混在一起处理
4. **文件命名不一致** — 有日期和无日期格式混用
5. **用户筛选效率低** — 逐个讨论 10 个候选，跳过率 ~60%

## 优化后的流程

```
① 饱和检查：读 content/topics/ → 构建已探索话题索引
② 合并搜索：1 个 librarian 检索中文+全球全源，附带已探索清单
③ 候选输出：≤ 10 条，每条标注 热度/类型/核心钩子
④ 单次候选表：一次性展示所有候选，用户逐条标记 要/跳过/合并到已有
⑤ 批量写入：新建 topic 文件 / 增量追加到已有文件
```

## 详细设计

### 1. 饱和检查（搜索前置）

- **操作**：搜索前读 `content/topics/` 目录下的所有 `.md` 文件名（排除 `old/` 子目录中的过期文件）
- **输出**：按文件名解析话题标题，构建"已探索话题索引"列表
- **用途**：传递给 librarian agent 作为已知话题过滤依据

### 2. 合并搜索（单 Agent 全源检索）

- **Agent**：1 个 librarian，一次性完成所有平台检索
- **平台覆盖**：
  - 中文：TrendsHub（知乎/微博/微博/36氪/掘金/InfoQ/少数派/B站/抖音/头条/爱范儿/澎湃/腾讯新闻）
  - 全球：BraveSearch（web/news/video）/ Tavily
- **输出格式**：统一的候选列表，内部分组（中文源/全球源），agent 自动做跨平台话题去重归并
- **去重规则**：同一事件的中英文报道合并为一条候选，标注各自的平台热度信号
- **已探索过滤**：prompt 中附带"已探索话题索引"，要求**优先提出新方向，避免重复返回已覆盖的话题**
- **事后匹配**：候选到齐后，Sisyphus 对比 topics/ 索引，高度相似命中的候选自动标记为 `[更新]` 而非新话题

### 3. 话题分类（自动标注）

每条候选自动标注类型标签：

| 标签 | 定义 | 判据 |
|------|------|------|
| `[热点]` | 时效 < 2 周，事件驱动 | 24-48h 内多个平台同时出现 |
| `[趋势]` | 活跃 2-4 周，发展阶段 | 跨周出现，有持续讨论 |
| `[范式]` | 持续 > 1 月，底层变化 | 行业/技术层面的结构性转变 |

### 4. 用户交互（单次候选表）

候选表格式：

```
┌────┬────────────────────────────────┬─────────┬────────┬──────────┐
│ #  │ 候选话题                        │ 热度    │ 类型   │ 你的标记  │
├────┼────────────────────────────────┼─────────┼────────┼──────────┤
│ 1  │ Claude Fable 5 下架...          │ ⭐⭐⭐⭐  │ [热点] │ [要/跳过] │
│ 2  │ AI Agent 爆发悖论（已存在）     │ ⭐⭐⭐   │ [趋势] │ [更新]   │
│ ... │                                │         │        │          │
└────┴────────────────────────────────┴─────────┴────────┴──────────┘
```

用户一次性看到所有候选，回复格式：`1 要 2 [更新] 3 跳过 4 要...`

候选表支持三种标记：
- `要` — 新建 topic 文件
- `[更新]` — 增量追加到已存在的话题文件，同步更新文件名时间戳
- `跳过` — 放弃此候选

### 5. 文件命名规范

**格式**：`{YYYYMMDD}-{中文主题名}.md`

拓扑文件结构：
```
content/topics/
├── 20260619-AI免费模式商业化困局.md
├── 20260621-AI Agent 爆发悖论.md     ← 更新后时间戳同步
├── archive/                          ← 已发表或 > 30 天
│   └── 20260501-旧话题.md
```

### 6. 增量更新规则

当饱和检查发现高度相似话题时：

- **操作**：读取已有 topic 文件 → 在文件尾追加新的内容区块（包含日期头 `## {YYYY-MM-DD} 更新`）
- **文件名**：时间戳同步更新为当前日期
- **保留原始内容**：不覆盖已有内容，确保每次更新的增量可见

## 关键变更对照

| 方面 | 现状 | 优化后 |
|------|------|--------|
| 搜索 agent | 2 个 librarian | 1 个 librarian |
| 深度研究 | 额外 5 个 librarian | 融合进主 agent |
| 饱和检查 | 无 | 搜索前读 topics/ 索引 |
| 话题分类 | 无 | 自动标注 `[热点]/[趋势]/[范式]` |
| 用户筛选 | 逐条讨论（10轮） | 单次候选表（1轮） |
| 文件命名 | 不一致 | `{YYYYMMDD}-{中文主题名}.md` |
| 话题重复 | 从零创建 | 增量追加到已有文件 |
| 归档 | 无 | `archive/` 目录 |
